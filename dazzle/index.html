About Our project

“Today the company obtain the screening questions of each job opportunity manually. For that, we built a scraper of screening questions called Project Mantis, once Mantis obtain the screening questions, we send them the information in a JSON format to be normalized the questions in the Torre’s format.”

Reduction of time

Now days the process of obtain all the screening questions for a job opportunity takes between 4 and 5 minutes. With Mantis the time will be reduces just to 14 seconds

Reduction of cost

As the process is carrying on freelancers then obtain each single screening question has a cost indeed, with Mantis the cost of obtain a single question is zero

Increase the cover of job opportunities

At this moment, Torre only can get access to a limit number of job offers per day. However, with Mantis the number will increase.

Clean code

Using the best practise that we have been learn over out ime in Holberton we create a software with the best practise in the industry

Easy to mantain

The software has been built in a easy and friendly way, helping the process of maintanence

Reduction of human error

The algorithm has been debugged, therefore the percentage of error is lower than the actual process

How the algorithm works?

Receive the links

Mantis receives an array of links belongs to each job opportunity.

Get actions

Depending of each link our algortihm checks at what webpage belongs the link and takes actions depending the caracteristics of each webpage and returning n HTML form.

Scraping the forms

Depending of each form we obtain each screening question and then we stored in a JSON.

Sending the JSON

Once the object has been created with all the screening questions then we send the object to be render in the torre format using retool.
